# Opportunities

This document identifies the meaningful opportunities revealed through the landscape, competitor analysis, user behaviour insights, and constraints. These opportunities represent areas where the product can create leverage, reduce friction, and deliver value that existing solutions do not.

Opportunities fall into four categories: experience, data, trust, and architecture.

## 1. Expereince OPpportunities

### 1.1. A map-first, answer-first interface

Parents don’t want to browse. They want the nearest usable facility.

Opportunity:

- A map that immediately surfaces the closest high‑quality options
- Clear visual cues (icons, colours, photos)
- One‑tap access to essential details

This reduces cognitive load and accelerates decision‑making.

---

### 1.2. One-handed, low-interaction design

Parents often have a baby in one arm and a phone in the other.

Opportunity:

- Thumb‑reachable controls
- Minimal typing
- Swipe and tap based flows
- Large touch targets

This is a differentiator, no competitor optimises for this.

---

### 1.2. Ultra-light contribution flow

Parents will contribute if it takes seconds.

Opportunity:

- One‑tap rating
- Quick photo upload
- Structured attributes instead of text reviews
- Offline contributions that sync later

This builds data density without friction.

---

### 1.4. Accessibility as a first-class feature

No competitor meaningfully addresses accessibility.

Opportunity:

- Step‑free access indicator
- Door width / pram space
- Changing table height
- Lighting and layout cues

This serves parents with prams, disabled parents, and carers; a large, underserved group.

---

## 2. Data Opportunities

### 2.1. Facility level granularity

Competitors focus on venues; parents care about the facility.

Opportunity:

- Each facility as a discrete entity
- Structured metadata (cleanliness, space, safety, amenities)
- Photos tied to the facility, not the venue

This is the core gap in the market.

---

### 2.2. Recency as a trust signal

Parents trust “updated 2 days ago” more than any rating.

Opportunity:

- Recency badges
- “Last checked” timestamps
- Highlighting recently updated facilities

This builds trust without heavy moderation.

---

### 2.3. Crowd-sourced freshness

Official data is insufficient and decays quickly.

Opportunity:

- Parent generated updates
- Lightweight validation (multiple recent contributions)
- Automated decay indicators (“stale data”)

This keeps the map alive.

---

### 2.4. Offline caching of local areas

Parents often search in poor‑signal environments.

Opportunity:

- Pre‑cache tiles and facility data
- Offline contributions
- Seamless sync when online

This is a competitive advantage.

---

## 3. Trust Opportunities

### 3.1. Photo-first trust model

Photos are the highest‑signal trust anchor.

Opportunity:

- Encourage photo uploads
- Show recent photos prominently
- Use photos to validate structured attributes

This reduces uncertainty and speeds decisions.

---

### 3.2. Transparent, simple ratings

Binary or 3‑point ratings outperform 5‑star systems.

Opportunity:

- “Good / OK / Avoid”
- Colour‑coded indicators
- No long reviews

This aligns with fast‑thinking behaviour.

---

### 3.3. Parent-to-parent credibility

Parents trust other parents more than institutions.

Opportunity:

- “Updated by a parent 3 days ago”
- No corporate tone
- No venue‑driven content

This positions the product as a community tool, not a directory.

---

## 4. Architectural Opportunities

### 4.1. PWA as a strategic choice

A PWA aligns with:

- Zero friction
- Offline capability
- Cross‑platform reach
- Lightweight updates

Opportunity:

- Instant access in moments of need
- No app‑store dependency
- Lower operational overhead

---

### 4.2. Minimal backend, maximmum resilience

A thin backend reduces:

- Cost
- Complexity
- Maintenance burden

Opportunity:

- Static hosting for most assets
- Simple API for contributions
- Caching and offline‑first patterns

This keeps the system robust and cheap to run.

---

### 4.3. Structured Data Model

A structured model enables:

- Fast filtering
- Clear visual cues
- Consistent entries
- Easy contribution flows

Opportunity:

- A schema that reflects real parent needs
- Predictable UX patterns
- High signal‑to‑noise ratio

---

## 5. Opportunity Summary

The strongest opportunities sit at the intersection of:

- Facility‑level data
- Photo‑first trust
- One‑handed, low‑cognitive‑load UX
- Offline‑friendly architecture
- Crowd‑sourced freshness
- Accessibility metadata

These opportunities define the shape of a product that is genuinely useful in the real world and meaningfully differentiated from existing solutions.
